Resumo do Projeto - Hackathon
Definição do Produto
O que é o produto?
É uma plataforma whitelabel de automação de chamadas por voz (Voice AI), projetada para que empresas possam configurar e escalar o contato com seus clientes. O sistema permite o agendamento de ligações, a personalização de agentes virtuais (voz e comportamento) e a captura de dados analíticos das conversas.
Qual problema ele soluciona?
Muitas empresas lutam para manter um contato proativo e personalizado com seus clientes sem escalar massivamente seus custos com call centers. Nossa solução ataca diretamente essa dor, automatizando o processo de chamadas de forma inteligente. Isso resulta em:
* Redução de Custo Operacional: Diminui a necessidade de mão de obra para ligações de rotina.
* Aumento de Eficiência: Capacidade de realizar milhares de chamadas simultâneas, 24/7.
* Inteligência de Dados: Transforma conversas em dados estruturados, gerando insights para o negócio.
* Escalabilidade: Permite que a empresa cresça sua base de clientes sem aumentar proporcionalmente sua equipe de atendimento.

Engenharia e Estratégia
Requisitos Funcionais do Projeto
Requisitos Funcionais (RFs) descrevem o que o sistema deve fazer. São as ações, as funcionalidades.
Módulo de Autenticação e Gestão da Empresa (RF-EMPRESA)
* RF01: O sistema deve permitir que um usuário (empresa) se cadastre e faça login.
* RF02: O usuário autenticado deve poder visualizar e atualizar as informações da sua empresa (nome, logo, cor primária para o whitelabel).
* RF03: O sistema deve permitir diferentes cargos/roles para usuários da mesma empresa (ex: Admin, Operador, Visualizador).
* RF04: O sistema deve controlar permissões baseadas no cargo do usuário.
Módulo de Gestão de "Sujeitos" (RF-SUJEITO)
* RF05: O sistema deve permitir o cadastro manual de um Sujeito, contendo, no mínimo, um número de telefone.
* RF06: O sistema deve permitir o cadastro em massa de Sujeitos (ex: importação de CSV).
* RF07: O sistema deve permitir a visualização de uma lista de todos os Sujeitos cadastrados.
* RF08: O sistema deve permitir a edição e exclusão de um Sujeito.
Módulo de Configuração do "Agente" (RF-AGENTE)
* RF09: O sistema deve permitir a criação de um novo Agente, definindo um nome para ele.
* RF10: O usuário deve poder definir o Objetivo principal do Agente (ex: "Vender Produto", "Agendar Reunião").
* RF11: O usuário deve poder configurar a Personalidade do Agente selecionando entre 3-4 opções pré-definidas ("Amigável", "Formal", "Persuasivo").
* RF12: O usuário deve poder selecionar a Voz do Agente entre 2-3 opções básicas (ex: "Masculina", "Feminina").
* RF13: O usuário deve poder adicionar Steps ao fluxo da conversa (mínimo 1, máximo 10 steps).
* RF14: Para cada Step, o usuário deve poder inserir um Título e o Texto Guia daquele step.
* RF15: O usuário deve poder reordenar os Steps usando botões "subir/descer".
* RF16: O usuário deve poder deletar um Step específico.
Módulo de Operação de Chamadas (RF-CALL)
* RF17: O usuário deve poder iniciar uma ligação para um Sujeito específico usando um Agente configurado.
* RF18: O usuário deve poder agendar o disparo de ligações para uma lista de Sujeitos, a cargo de um Agente específico, para um horário determinado.
Módulo de Análise e Feedback (RF-ANALISE)
* RF19: O sistema deve armazenar um histórico de todas as conversas realizadas.
* RF20: Para cada conversa, o sistema deve exibir: Sujeito, Agente utilizado, Status (concluída, falhou, etc.), e o histórico da conversa (transcrição).
* RF21: O sistema deve gerar e exibir um Resumo em texto da conversa ao seu final.
* RF22: O sistema deve exibir em um painel (dashboard) métricas gerais, como: total de ligações, taxa de sucesso dos objetivos, etc.
* RF23: O sistema deve analisar e pontuar a conversa com base em Pesos (análise de sentimento/emoção).
* RF24: O sistema deve identificar e sinalizar em qual "step" (tópico) da conversa o Agente teve mais dificuldade.

Matriz de Impacto x Esforço
Quadrante I: Alto Impacto, Baixo Esforço (Prioridade)
* Autenticação simples (RF01): Login básico com JWT.
* Whitelabel básico (RF02): Upload de logo + seletor de cor primária.
* Cadastro manual de Sujeitos (RF05): Formulário simples com nome e telefone.
* Criar Agente básico (RF09, RF10): Nome + Objetivo.
* Personalidade + Voz (RF11, RF12): Opções pré-definidas (dropdowns).
* Gerenciar Steps (RF13, RF14): Adicionar e editar steps (título + texto).
* Reordenar Steps (RF15): Botões para mover os steps.
* Realizar UMA ligação (RF17): Integração com API de voz e LLM.
* Visualizar histórico (RF19, RF20): Lista de conversas com transcrição.
Quadrante II: Alto Impacto, Alto Esforço
* Sistema de Cargos/Roles (RF03, RF04): Controle de permissões por cargo na empresa.
* Gerar Resumo da conversa (RF21): Chamada à API de LLM para resumir a conversa.
* Dashboard com analíticas (RF22): Exibição de métricas e gráficos.
* Agendamento de ligações em massa (RF18): Sistema de fila e workers no backend.
* Steps com drag-and-drop: Melhorar a experiência de usuário para reordenar os steps.
Quadrante III: Baixo Impacto, Baixo Esforço
* Deletar Steps (RF16): Botão de lixeira para remover steps.
* CRUD completo de Sujeito/Agente (RF08): Adicionar botões de editar e deletar.
* Filtros no painel: Filtrar conversas por data ou status.
* Validações de formulário: Melhorar a UX com validações nos formulários.
Quadrante IV: Baixo Impacto, Alto Esforço
* Análise de emoção e pontuação (RF23): Exige APIs de terceiros específicas ou NLP avançado.
* Feedback coletado com base em falhas de steps (RF24): Requer lógica sofisticada para mapear falhas.
* Importação de CSV (RF06): Pode ser uma armadilha de tempo devido ao tratamento de parsing e erros.
